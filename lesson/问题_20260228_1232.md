# 问题标题
2026-02-28 12:32，计时弹窗按钮仍被中文全键盘顶部候选栏遮挡

# 问题描述
在将底部操作栏改为贴近键盘后，计时弹窗使用中文全键盘时，取消/确定按钮仍有一部分被系统键盘顶部候选栏覆盖；燃油弹窗的数字键盘场景则基本正常。

# 根本原因分析
BottomSheet 外层为了弱化 iOS 数字键盘圆角，整体向下压入键盘顶部一段固定距离。这个策略在数字键盘下问题不明显，但在带候选栏的中文全键盘下，键盘顶部是实心内容区域，结果固定压入量直接覆盖了按钮。

# 解决方案与步骤
1. 保留外层 BottomSheet 的统一键盘过渡策略，不破坏燃油页当前效果。
2. 仅在计时弹窗内部检测键盘是否弹起。
3. 键盘出现时，在底部操作栏下方补一个与 `keyboardTopOverlap` 等高的占位，让按钮整体上移，避开被键盘覆盖的区域。

# 改进启示
同一套键盘过渡策略未必适用于所有表单。对长表单或文本键盘场景，应允许在具体页面内做局部补偿，而不是强行用一套全局位移覆盖所有输入类型。
