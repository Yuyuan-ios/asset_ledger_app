# 问题标题
2026-02-28 20:28，项目详情弹窗顶部间距与设备单价区同行对齐不一致

# 问题描述
项目详情弹窗中，标题下首行内容与其它弹窗的分割线间距不一致；同时“设备单价”和第一台设备信息没有真正同行，且多设备场景仍需保留“批量修改”。

# 根本原因分析
1. 项目详情弹窗仍沿用了 BottomSheet Shell 的默认内容 padding，导致顶部视觉间距比其它已收口的弹窗更大。
2. “设备单价”标签额外下沉，设备列表首行也有对称上下 padding，导致第一行内容与标签没有对齐到同一基线。

# 解决方案与步骤
1. 在项目详情弹窗调用处将 `contentPadding` 改为 `EdgeInsets.zero`。
2. 在内容组件内部仅补回统一的左右 padding，避免重新引入顶部额外间距。
3. 去掉“设备单价”标签的顶部偏移。
4. 让第一台设备行从 `top: 0` 开始，与“设备单价”真正同行；后续设备再向下排列。
5. 保留“设备数大于 1 时显示批量修改”的逻辑。
6. 将项目详情弹窗进度条颜色改为与账户主页一致的绿色。

# 改进启示
统一弹窗节奏时，必须同时检查调用层 padding 和内容层布局；只改一层会留下“看起来像没改”的视觉残差。
