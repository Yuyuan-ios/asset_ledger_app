# 问题标题
2026-02-28 20:08，设备色板语义不够清晰且已实收绿色仍偏浅

# 问题描述
账户页总览设备颜色虽然当前未使用红绿，但仅以匿名数组存在，后续容易被误改；同时“已实收”绿色还需要再深一档。

# 根本原因分析
1. 设备色板缺少明确的命名与约束说明，维护者无法一眼看出这是需要避开红绿的设备专用色板。
2. 共享的 `chartIncome` 颜色仍偏亮，在白底界面上力度不足。

# 解决方案与步骤
1. 将账户页设备色板拆成命名颜色常量。
2. 给该组颜色增加“冷色系设备色板，避免误用红/绿”的语义注释。
3. 保持 `overviewChartPalette` 继续复用这些命名常量。
4. 将 `TimingColors.chartIncome` 再压深一档，使进度条和饼图已实收颜色同步增强。

# 改进启示
需要长期稳定的颜色体系，不应只保留色值数组；命名和约束说明本身就是设计系统的一部分。
