# 问题标题
2026-02-28 12:15，底部弹窗操作栏与系统键盘间距在不同键盘类型下视觉不一致

# 问题描述
计时弹窗与燃油弹窗共用同一套底部操作栏，但在 iOS 数字键盘和中文全键盘下，取消/确定按钮与系统键盘之间的视觉间距看起来不一致，影响界面一致性。

# 根本原因分析
底部操作栏在键盘弹起时仍然叠加了固定底部留白，导致按钮区与键盘之间保留额外间距。不同系统键盘的高度、顶部圆角和候选栏样式不同，这个固定留白会被放大成明显的视觉差异。

# 解决方案与步骤
1. 调整底部操作栏的 padding 计算逻辑。
2. 键盘弹起时不再追加 `footerBottom` 和额外键盘间距，底部 padding 直接置为 `0`。
3. 键盘未弹起时，仍保留原有底部安全区与基础留白，避免影响常规布局。

# 改进启示
跨输入法/跨键盘类型的 UI 对齐，不应依赖固定视觉间距常量。更稳妥的策略是键盘弹起时让关键操作区直接贴边，再通过外层容器统一控制过渡。
