# 问题标题
2026-02-28 12:53，账户页筛选弹窗错误缩小高度导致比例失真与底部溢出

# 问题描述
为让账户页筛选弹窗底部按钮贴底，误将弹窗整体高度改小，导致它与计时、燃油弹窗的页面占比不一致，并出现底部溢出提示。

# 根本原因分析
问题的目标是“在同样高度壳体内让操作区贴底”，不是“缩小壳体高度”。直接降低 `initialHeightFactor` 改变了容器比例，反而破坏了与其他通用弹窗的一致性，并压缩了底部可用空间。

# 解决方案与步骤
1. 移除账户页筛选弹窗上单独设置的 `initialHeightFactor`。
2. 恢复使用通用 BottomSheet 的默认高度比例，与计时、燃油弹窗保持一致。
3. 保留 `scrollable: false + Column + Spacer + BottomActionBar` 的布局，让底部操作区在高壳体内固定贴底。

# 改进启示
要区分“容器尺寸问题”和“内部布局问题”。当目标是让按钮贴底时，应优先修正内部约束，而不是通过改变外层高度去掩盖布局缺陷。
