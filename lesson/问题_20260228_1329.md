# 问题标题
2026-02-28 13:29，燃油弹窗在统一补偿后仍与计时弹窗存在键盘间距差异

# 问题描述
燃油页弹窗在中文和数字输入法下，键盘顶部候选栏与确定按钮之间仍比计时弹窗多出一段间距；账户筛选与计时弹窗表现一致。

# 根本原因分析
在内部布局已对齐后，燃油弹窗与计时弹窗在外层 `showModalBottomSheet` 调用上仍存在参数差异。燃油页使用 `useSafeArea: true`，而计时页使用 `useSafeArea: false`，这会让路由层的系统避让策略不同，导致最终按钮与键盘的视觉距离不一致。

# 解决方案与步骤
1. 将燃油页的 `showModalBottomSheet` 参数与计时页完全对齐。
2. 把 `useSafeArea` 改为 `false`。
3. 显式补齐 `initialHeightFactor` 与 `radius` 参数，确保外层壳调用方式一致。

# 改进启示
当多个页面共用统一弹窗壳时，仅统一壳内布局还不够；路由层的调用参数也必须一致，否则同一套内部组件仍可能因为上层策略差异表现不同。
