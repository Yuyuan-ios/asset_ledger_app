# 问题标题
2026-02-28 15:06，记录列表头像需要进一步放大到 50x50

# 问题描述
在将记录列表头像调整到 40x40 后，仍希望进一步提升头像识别度与品牌区分度，统一放大到 50x50。

# 根本原因分析
当前头像尺寸虽已统一到同一 token，但视觉上仍偏保守，在列表信息密度允许的前提下，可以通过进一步放大头像增强快速识别能力。

# 解决方案与步骤
1. 仅调整共用记录头像尺寸 token。
2. 将 `TimingTokens.recordAvatarSize` 从 40 调整为 50。
3. 由于燃油页头像已对齐到同一 token，本次修改会同步影响计时页与燃油页。

# 改进启示
当同类元素已经被正确收口到统一 token 后，后续视觉微调的成本会显著下降。统一入口的价值，就体现在这种“一次改动、全局同步”的调整效率上。
