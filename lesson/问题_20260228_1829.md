# 问题标题
2026-02-28 18:29，账户页项目详情弹窗顶部状态信息重排

# 问题描述
项目详情弹窗顶部右侧原本显示日期，设备单价区域下方再单独显示“剩余/应收”金额文本。用户希望将更关键的剩余金额直接放到标题行，并把下方余额文案改为收款进度条。

# 根本原因分析
原布局把关键财务状态拆散在两个位置，日期占用了标题区的主视线位置，而下方文本对“收款进度”的表达也不够直观。

# 解决方案与步骤
1. 将标题行右侧日期改为 `余：剩余/应收`。
2. 计算已收金额与进度比例。
3. 用橙色 `LinearProgressIndicator` 替换原余额文本行，直观显示收款进度。

# 改进启示
详情页顶部应优先展示对当前决策最重要的状态信息，次级信息可以后置或弱化，避免占用主视觉。
