# 问题标题
2026-02-28 13:22，账户筛选弹窗底部按钮左右边距与计时弹窗不一致

# 问题描述
账户页筛选弹窗的取消/确定按钮相比计时弹窗更靠中间，左右距离页面边缘更大，视觉对齐不一致。

# 根本原因分析
账户筛选弹窗将整个内容区（包括 `BottomActionBar`）包在一层左右 `16` 的外层 `Padding` 内，而 `BottomActionBar` 本身又有自己的左右边距。两层水平间距叠加后，导致按钮整体向内收缩。

# 解决方案与步骤
1. 保留输入区的左右内边距，维持字段宽度与现有布局。
2. 将 `BottomActionBar` 移出外层水平 `Padding`。
3. 让底部按钮只使用组件自身的统一左右边距，从而与计时弹窗对齐。

# 改进启示
复用统一组件时，父容器的额外包裹层同样会改变最终视觉结果。要做真正一致的对齐，不能只看组件本身，还要检查外层布局是否重复叠加了同类间距。
