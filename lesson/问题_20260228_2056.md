# 问题标题
2026-02-28 20:56，项目详情弹窗单价区需按草图重排

# 问题描述
项目详情弹窗的单价区需要严格按草图呈现：`设备单价` 必须和第一台设备信息处于同一行，左侧第二行才是 `批量修改`，并且标签到设备列的距离应可调。

# 根本原因分析
现有布局虽然做了列对齐，但标签区与设备区之间没有独立的间距 token，且结构没有完全固定为“左两行标签 + 右多行设备”的稳定关系。

# 解决方案与步骤
1. 新增 `projectDetailRateLabelGap`，专门控制“设备单价”和设备列之间的距离。
2. 将左侧固定为 `设备单价 / 批量修改` 的两行标签列。
3. 右侧保留设备列表多行，第一行与“设备单价”同行起始。
4. 将收款记录的左侧对齐计算同步改为复用新的间距 token。

# 改进启示
当布局有明确的视觉锚点时，应把“列间距”独立成 token，而不是混在其他缩进参数里，避免后续微调牵一发而动全身。
