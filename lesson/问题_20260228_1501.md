# 问题标题
2026-02-28 15:01，计时与燃油记录列表的头像与文字尺寸未对齐到最新设计稿

# 问题描述
记录列表在计时页和燃油页中使用了相近结构，但燃油页头像尺寸没有被明确约束到与计时页一致的 token，且两页记录文字仍停留在旧字号，与 Figma 中“头像 40x40、文本 14”不一致。

# 根本原因分析
1. 计时页头像尺寸由 `TimingTokens.recordAvatarSize` 控制，燃油页头像却由 `leadingBuilder` 直接返回组件，缺少同一尺寸约束。
2. 记录行标题、副标题和右侧数值统一依赖 `TimingTokens`，仍使用旧的 13 号字。

# 解决方案与步骤
1. 将 `TimingTokens.recordAvatarSize` 从 38 调整为 40。
2. 将 `recordTitleFontSize`、`recordSubTitleFontSize`、`recordValueFontSize` 从 13 调整为 14。
3. 在燃油页 `leadingBuilder` 中用 `SizedBox` 包裹头像，并显式传入与 `TimingTokens.recordAvatarSize` 对齐的宽高和半径。

# 改进启示
当多个页面共享同一视觉元素时，仅共享字号 token 不够，尺寸约束也必须落到统一入口。否则即使名义上“共用组件”，实际视觉仍会悄悄分叉。
