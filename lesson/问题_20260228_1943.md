# 问题标题
2026-02-28 19:43，底部弹窗标题分割线到内容区间距来源不统一

# 问题描述
计时弹窗、燃油弹窗、账户弹窗虽然共用同一个 BottomSheet Shell，但部分内容组件又额外叠加了首屏顶部 padding，导致标题分割线到内容的实际视觉间距不一致。

# 根本原因分析
外层 `AppBottomSheetShell` 已通过 `dividerToContentGap` 统一定义了分割线到内容区的基准距离，但计时弹窗和账户筛选弹窗又在内容内部追加了顶部留白，形成双重间距来源。

# 解决方案与步骤
1. 将计时弹窗内容区首屏顶部 padding 从 `10` 改为 `0`。
2. 将账户筛选弹窗输入区首屏顶部 padding 从 `12` 改为 `0`。
3. 保持燃油弹窗不变，因为其内容区顶部原本已为 `0`。

# 改进启示
弹窗的结构性间距应只有一个来源。标题分割线到内容区的距离应统一由 shell 控制，内容组件只负责内部字段之间的间距。
