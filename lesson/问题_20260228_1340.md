# 问题标题
2026-02-28 13:40，底部弹窗结构长期分散在多个文件与散装 token 中

# 问题描述
底部弹窗的标题栏、分割线、底部操作区、键盘补偿和结构 token 分散在多个文件中，导致页面层重复拼装 footer，出现参数叠加、边距错位和重复修补。

# 根本原因分析
`AppBottomSheetShell` 只承担“容器壳”职责，而底部操作区和键盘补偿被拆在外部页面与独立组件中。随着多个弹窗都依赖相同底部结构，职责边界已经失衡，违反 DRY，也放大了布局不一致风险。

# 解决方案与步骤
1. 新增 `BottomSheetTokens`，集中管理弹窗壳与底部操作区结构 token。
2. 升级 `AppBottomSheetShell`，支持可选 footer（取消/确定）、标题下分割线和内建键盘补偿。
3. 计时、燃油、账户筛选改为直接使用 shell 的 footer 配置，不再在内容层手工拼底部按钮。
4. 删除旧的 `bottom_action_bar.dart` 与 `keyboard_overlap_compensation.dart`。
5. 清理被迁移出 `SheetTokens` 的旧结构 token。

# 改进启示
当某类 UI 结构已经在多个页面重复出现时，继续靠页面拼装只会积累“局部看似可用、整体越来越乱”的技术债。应该把这类结构提升为有明确边界的可选壳能力，并配套把 token 一起收口。
