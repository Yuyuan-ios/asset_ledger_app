# 问题标题
2026-02-28 14:38，底部 toast 提示框仍偏大，文案存在轻微“压视觉”感

# 问题描述
在调整为居中自适应 toast 后，提示框整体仍稍显偏大，尤其是短文案（如“已保存”）时，容器体积和字体都还有些重。

# 根本原因分析
toast 的圆角、内边距和字号仍偏向常规按钮级别，导致短提示虽然不再铺满底部，但整体块感仍略强。

# 解决方案与步骤
1. 缩小 toast 圆角半径。
2. 收紧水平和垂直内边距，让提示框更贴合文案尺寸。
3. 将文字字号从更显眼的尺寸下调一档，保留清晰度但减轻压感。

# 改进启示
短提示的体量应该服从内容长度。对“已保存”这类单句反馈，容器和字体只要足够看清即可，略小一档通常更符合“轻提示”的气质。
