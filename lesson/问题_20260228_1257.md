# 问题标题
2026-02-28 12:57，账户页筛选弹窗在中文全键盘下底部按钮再次与键盘重叠

# 问题描述
账户页筛选弹窗改为统一底部操作区后，在中文全键盘场景下，取消/确定按钮再次与系统键盘顶部候选栏发生重叠，出现与计时弹窗此前相同的问题。

# 根本原因分析
通用 BottomSheet 外层仍会在键盘弹起时向下压入 `keyboardTopOverlap` 的固定距离，用于弱化 iOS 键盘圆角。这个策略对带候选栏的中文全键盘同样会把底部操作区覆盖一部分；筛选弹窗内部尚未像计时弹窗一样做局部反向补偿。

# 解决方案与步骤
1. 不改通用 BottomSheet，避免影响燃油页和其他已稳定弹窗。
2. 在账户页筛选弹窗内部检测键盘是否弹起。
3. 键盘出现时，在 `BottomActionBar` 下方补一个与 `keyboardTopOverlap` 等高的占位。
4. 让底部操作区整体上移，避开键盘顶部重叠区。

# 改进启示
凡是复用统一底部操作栏且包含文本输入的弹窗，只要外层仍采用固定键盘 overlap 策略，都应在页面内预留局部补偿点；否则同类重叠问题会在不同弹窗反复出现。
