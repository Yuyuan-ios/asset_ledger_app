# 问题标题
2026-02-28 21:01，收款记录误对齐到设备编号列

# 问题描述
项目详情弹窗中，收款记录内容被对齐到了设备编号列起点，导致相比“设备单价”标签左边界明显右移，不符合预期。

# 根本原因分析
收款记录左侧缩进错误复用了设备列的对齐计算（标签列宽 + 列间距），而用户实际要求是与“设备单价”标签本身的左边界对齐。

# 解决方案与步骤
1. 去掉收款记录对设备编号列起点的复用计算。
2. 将左侧缩进直接改为 `projectDetailRateTitleLeftInset`。
3. 保持右侧编辑/删除按钮位置不变。

# 改进启示
同一个页面内存在多条参考线时，必须先明确“对齐对象”是哪一条线；否则即使数值正确，也会对齐到错误目标。
