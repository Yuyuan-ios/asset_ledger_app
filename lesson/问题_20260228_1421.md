# 问题标题
2026-02-28 14:21，底部提示条过宽且文案未居中，提示感受生硬

# 问题描述
页面底部的“已保存”等提示当前使用默认 `SnackBar`，会出现整条铺满底部的深色横条，文字偏左，视觉上显得过重，也不符合“轻提示”的交互预期。

# 根本原因分析
项目中多数页面直接调用默认 `SnackBar`，没有统一 toast 样式。默认行为会占据整行宽度，内容也按常规消息布局显示，导致提示框无法根据文案长度自适应。

# 解决方案与步骤
1. 升级统一 `AppToast` 实现，改为浮动型提示。
2. 使用透明 `SnackBar` 外壳，只渲染中间的自适应内容容器。
3. 提示容器按文本内容包裹，内边距固定，文字水平居中。
4. 将多个页面的直接 `SnackBar` 调用收敛到 `AppToast.show()`。

# 改进启示
轻提示不应该像错误拦截条一样占满底部。对“已保存”这类短反馈，最合适的是小而明确、自动消失、视觉居中的提示，让用户收到反馈但不被打断。
