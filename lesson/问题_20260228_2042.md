# 问题标题
2026-02-28 20:42，多设备单价行需要表格化对齐

# 问题描述
项目详情弹窗在多设备场景下，设备名、价格、修改按钮虽然同在一块，但每行没有形成稳定列对齐，不符合期望的整齐布局。

# 根本原因分析
原布局依赖自适应宽度和普通 `Row` 分配空间，价格列与操作列没有固定列宽，导致多行之间难以形成清晰的垂直对齐线。

# 解决方案与步骤
1. 将“批量修改”移到右上方单独一行。
2. 下方改为“左侧固定标签 + 右侧设备列表”的两列结构。
3. 右侧设备列表内部为每行建立固定价格列和固定操作列。
4. 多台设备时，所有行共享同一列宽，保证设备名、价格、修改按钮上下对齐。

# 改进启示
当一组信息需要跨多行比较时，应优先使用稳定列宽，而不是完全依赖自适应布局。
