# 问题标题
2026-02-28 12:45，账户页筛选弹窗虽然接入底部操作栏但仍未形成“贴底”布局

# 问题描述
账户页筛选弹窗引入 `BottomActionBar` 后，取消/确定按钮仍停留在内容中部，下方出现大块空白，视觉上仍与计时/燃油弹窗的底部固定操作区不一致。

# 根本原因分析
仅替换按钮组件并不能形成统一交互模式。筛选弹窗仍使用默认可滚动内容流布局，`BottomActionBar` 只是普通内容节点，没有被布局系统推到底部。

# 解决方案与步骤
1. 将筛选弹窗的 `AppBottomSheetShell` 改为 `scrollable: false`。
2. 让弹窗内容使用占满可用高度的 `Column`。
3. 在“清空”按钮和 `BottomActionBar` 之间插入 `Spacer()`。
4. 让取消/确定真正固定在弹窗底部，形成与计时/燃油一致的底部操作区模式。

# 改进启示
统一组件只是第一步，统一布局约束才是关键。只换同一个按钮组件但不调整父布局，视觉和交互仍然会偏离预期。
