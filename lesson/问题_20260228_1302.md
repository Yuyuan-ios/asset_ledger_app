# 问题标题
2026-02-28 13:02，多个底部弹窗重复实现键盘重叠补偿逻辑

# 问题描述
计时弹窗和账户筛选弹窗都需要在中文全键盘场景下，针对 BottomSheet 外层的键盘 overlap 做局部补偿。此前每个弹窗都在页面内部手写一次键盘检测和占位逻辑，存在重复代码与后续遗漏风险。

# 根本原因分析
键盘 overlap 补偿是一个跨弹窗的通用布局规则，但当前实现分散在具体页面中，违背 DRY，也提高了后续新增弹窗时重复踩坑的概率。

# 解决方案与步骤
1. 新增可复用组件 `KeyboardOverlapCompensation`。
2. 将“键盘弹起时补一个 `keyboardTopOverlap` 高度占位”的逻辑收口到统一组件内。
3. 计时弹窗与账户筛选弹窗改为直接复用该组件，不再各自手写键盘检测与 `SizedBox`。

# 改进启示
一旦同一 UI 修复逻辑出现第二次，就应该考虑抽象。布局补偿这类横切关注点不该散落在多个页面里，统一封装后更容易保持行为一致。
