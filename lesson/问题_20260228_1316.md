# 问题标题
2026-02-28 13:16，燃油弹窗接入键盘补偿后按钮与键盘之间仍有额外间距

# 问题描述
燃油页弹窗在中文和数字输入法场景下，取消/确定按钮与系统键盘顶部之间仍有明显间距，而计时弹窗和账户筛选弹窗表现正常。

# 根本原因分析
燃油弹窗在 `AppBottomSheetShell` 上仍使用默认 `contentPadding`，其中包含底部内边距；与此同时，弹窗内部又已经接入了 `KeyboardOverlapCompensation`。两层底部留白叠加后，导致按钮被整体再次上推。

# 解决方案与步骤
1. 对齐燃油弹窗与计时弹窗的 BottomSheet 参数。
2. 在燃油页打开弹窗时，为 `AppBottomSheetShell` 显式传入 `contentPadding: EdgeInsets.zero`。
3. 保留内部 `BottomActionBar + KeyboardOverlapCompensation`，避免重复底部间距。

# 改进启示
统一组件后，外层壳参数也必须同步统一。只统一内部布局而忽略外层默认 padding，仍然会出现“看起来像同一套，实际间距不一致”的问题。
