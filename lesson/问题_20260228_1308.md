# 问题标题
2026-02-28 13:08，账户页筛选入口按钮冗余且燃油弹窗未统一键盘补偿

# 问题描述
账户页在已筛选状态下同时显示“取消筛选”和“已筛选”两个入口，用户需要在相近位置做二次判断，增加操作成本。同时，燃油弹窗尚未接入统一的键盘重叠补偿组件，底部表单弹窗行为仍未完全收敛。

# 根本原因分析
1. 已筛选状态的交互目标已经明确是“撤销筛选”，保留额外的“已筛选”状态按钮属于冗余交互。
2. `KeyboardOverlapCompensation` 已抽出，但燃油弹窗仍沿用旧实现，没有接入统一补偿层。

# 解决方案与步骤
1. 账户页在有生效筛选时只显示“取消筛选”按钮，不再显示“已筛选”按钮。
2. 无筛选时仍显示“筛选”按钮，保持原始入口。
3. 燃油弹窗在 `BottomActionBar` 下方接入 `KeyboardOverlapCompensation`，与计时弹窗、账户筛选弹窗统一。

# 改进启示
减少用户输入成本最有效的方式之一，是在状态明确时只保留最相关的动作。对于重复出现的键盘补偿规则，应尽快让所有同类弹窗接入统一组件，避免行为碎片化。
