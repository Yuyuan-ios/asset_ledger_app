# 问题标题
2026-02-28 12:49，账户页筛选弹窗使用统一底部操作栏后仍显得“未贴底”

# 问题描述
账户页筛选弹窗已经切换为统一的底部操作栏布局，但视觉上仍然像操作区停留在中部，下方有大量空白，用户感知上并没有形成贴底效果。

# 根本原因分析
筛选弹窗继续沿用通用 BottomSheet 的默认高度比例（约 92% 屏高）。对于只有单个输入框的小型弹窗，这个高度过大，即使底部操作栏在弹窗底部，也会因为壳体过高而显得不在底部。

# 解决方案与步骤
1. 仅针对账户页筛选弹窗调小 BottomSheet 高度比例。
2. 将 `initialHeightFactor` 改为更紧凑的高度，减少无意义留白。
3. 将 `contentPadding` 设为 `EdgeInsets.zero`，避免壳体默认内边距与内部自定义 padding 叠加，保持布局可控。

# 改进启示
通用壳组件的默认尺寸不应被机械复用到所有场景。小型确认/筛选弹窗应使用紧凑高度，否则即使内部布局正确，也会因为容器过高造成“视觉上不贴底”的错觉。
