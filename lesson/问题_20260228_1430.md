# 问题标题
2026-02-28 14:30，底部 toast 提示背景偏重，轻提示感不足

# 问题描述
统一后的底部 toast 已经改为自适应宽度和居中展示，但背景色仍偏深，视觉分量略重，容易让短提示显得过于突兀。

# 根本原因分析
toast 容器背景使用了较深且不透明的深棕色，虽然可读性足够，但对于“已保存”这类轻反馈，仍然会给人偏重的压迫感。

# 解决方案与步骤
1. 仅调整 toast 背景色，不改位置、圆角和文字。
2. 将背景改为更浅、带透明度的暖棕色。
3. 保持文字对比度，继续确保可读性。

# 改进启示
轻提示的重点是“被看见但不打断”。背景色只要足够承托文字即可，过深会让次级反馈显得像主警示。
