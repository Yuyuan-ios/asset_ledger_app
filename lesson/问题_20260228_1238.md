# 问题标题
2026-02-28 12:38，计时弹窗首个设备输入框标签被裁切且账户筛选弹窗底部操作区样式不统一

# 问题描述
计时弹窗顶部第一个设备选择框紧贴分割线，浮动标签“设备编号”被遮挡。与此同时，账户页筛选弹窗虽然复用了通用 BottomSheet 壳，但底部按钮仍是手写布局，没有与计时/燃油页统一到底部操作栏模式。

# 根本原因分析
1. 计时弹窗内容区顶部内边距不足，首个带浮动标签的输入控件没有留出标签悬浮所需的上方空间。
2. 账户筛选弹窗只复用了外层 BottomSheet 壳，没有复用统一的底部操作栏组件，导致键盘联动和按钮布局策略分叉。

# 解决方案与步骤
1. 为计时弹窗滚动内容增加顶部留白，让设备选择框的浮动标签完整显示。
2. 在账户页引入 `BottomActionBar`。
3. 将筛选弹窗底部改为统一的取消/确定操作栏，“清空”保留为独立次级按钮。
4. 去掉筛选弹窗底部手工叠加的 `viewInsets.bottom`，交给统一底部操作栏处理键盘联动。

# 改进启示
复用不能只停留在外壳层。只要是同类弹窗交互，底部操作区和键盘联动也应统一，否则视觉和行为会逐步分叉，后续修一个页面就会漏另一个页面。
