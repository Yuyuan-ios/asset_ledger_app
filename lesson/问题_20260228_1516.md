# 问题标题
2026-02-28 15:16，账户页项目卡片缺少结构化 token，调节成本高

# 问题描述
账户页项目卡片的高度与视觉层级由多处散落的内边距、字号和间距共同决定，但这些数值直接写在组件里。后续想微调卡片体积或文字密度时，需要反复进入组件结构查找，调节成本高且容易漏项。

# 根本原因分析
项目卡片虽是账户页核心信息单元，但其布局参数没有被收口到领域 token，导致结构样式与业务组件耦合，违背“尺寸由 token 管理”的原则。

# 解决方案与步骤
1. 在 `AccountTokens` 中新增项目卡片专属结构 token。
2. 收口以下参数：
   - 卡片间距
   - 卡片内边距
   - 项目名、日期、标签、状态字号
   - 标题与日期间距
   - 各信息区块间距
   - 进度条上方间距
   - 进度条高度
3. 将 `account_project_list_pattern.dart` 中的硬编码数值替换为对应 token。

# 改进启示
真正决定卡片高度的往往不是一个“height”，而是一组结构参数。把这些参数完整收口后，布局才变得可预测、可复用、可低成本微调。
