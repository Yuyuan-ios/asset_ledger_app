# Agent 标准工作流

本仓库默认采用以下流程：只读审计 -> 实现 -> 验证 -> 报告 -> 人类确认 -> commit。

## 1. 只读审计

- 明确任务范围、限制、验收条件。
- 读取 `AGENTS.md` 和相关领域文档。
- 检查代码、文档、分支或 diff，但不修改文件。
- 输出发现、风险、待确认项和建议。

## 2. 实现

- 只改任务要求覆盖的文件。
- 优先沿用现有代码结构、命名、测试方式和脚本。
- 不修改业务逻辑、UI、数据库 schema 或测试断言，除非任务明确要求。
- 发现范围外问题时记录为遗留问题，不顺手扩展。

## 3. 验证

- 小范围文档或脚本改动至少运行 fast 检查。
- 涉及共享逻辑、数据库、导入导出、统计口径时运行 full 检查。
- 验证失败时先判断是否由本次改动引入，并在报告中给出证据。

## 4. 报告

报告应包含：

- 修改文件列表。
- 每个关键文件的作用。
- 实际运行的命令和结果。
- 是否修改业务代码。
- 遗留问题和待确认项。

## 5. 人类确认

未得到明确要求前，不执行 commit、push、删除分支或清理 worktree。

## 6. Commit

得到明确确认后再提交。提交前重新检查 `git status --short`，确认只包含本次任务相关改动。
