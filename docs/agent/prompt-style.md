# Prompt 规范

本项目的 agent prompt 应完整、可复制、阶段化，并避免重复前置检查。

## 推荐结构

- 目标：说明要完成什么。
- 范围：说明允许新增、修改或审计哪些文件。
- 限制：说明禁止修改的区域，例如业务逻辑、UI、schema、测试断言。
- 验证：列出必须运行的命令。
- 最终报告：列出报告必须包含的信息。

## 写法规则

- 用清晰的阶段描述任务，例如“只读审计 -> 实现 -> 验证 -> 报告”。
- 前置检查只写一次，不在每个小节重复。
- 不主动注入分支名、commit hash、worktree 要求，除非任务明确要求。
- 不把待确认的产品细节写成事实。
- 对只读审计任务明确写出“只读，不修改、不格式化、不提交”。
- 对实现任务明确写出允许和禁止修改的范围。

## 输出偏好

- 审计报告先列问题，再列总结。
- 实现报告列出修改文件、验证结果、是否改业务代码、遗留问题。
- 中文产品文案保持一致，具体规则见 `docs/product/ui-copywriting.md`。
