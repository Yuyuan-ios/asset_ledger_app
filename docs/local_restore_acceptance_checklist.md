# 本地覆盖恢复功能验收清单

本文档用于手工验收机账通本地覆盖恢复功能。验收范围仅包含本地 JSON 备份的预检、确认、自动备份、完整覆盖恢复、事务回滚和页面刷新。

当前约束：

- 恢复模式：完整覆盖恢复。
- 当前数据库版本：`AppDatabase.schemaVersion = 9`。
- 只允许 `meta.schema_version == 9` 的备份执行真实恢复。
- 本地恢复前会自动导出当前本机数据备份。
- 不支持合并恢复、追加恢复、部分表恢复。

## 1. 测试前准备

测试环境：

- 使用测试机或模拟器。
- 不使用真实重要数据。
- 测试前确认 App 可正常启动，设备页、计时页、燃油页、维保页、账户页可访问。
- 测试前记录当前 App 版本和构建号。

基础测试数据：

- 准备至少 2 台设备。
- 准备至少 2 条计时记录。
- 准备至少 1 条油费记录。
- 准备至少 1 条维修记录。
- 准备至少 1 条收款记录。
- 准备至少 1 条项目相关设置，例如项目设备单价覆盖。

备份准备：

1. 在基础测试数据准备完成后，进入账户中心。
2. 点击“手动本地备份”。
3. 导出一份本地备份，记为备份 A。
4. 记录备份 A 的文件名、导出时间和摘要数量。
5. 使用“本地恢复”选择备份 A，确认可以预览。

## 2. 正常恢复测试

目标：验证同版本备份可以完整覆盖恢复，并且恢复后数据回到备份 A 状态。

步骤：

1. 确认备份 A 已导出。
2. 在 App 中修改部分业务数据：
   - 新增或删除 1 台设备。
   - 新增、编辑或删除 1 条计时记录。
   - 新增、编辑或删除 1 条油费记录。
   - 新增、编辑或删除 1 条维修记录。
   - 新增、编辑或删除 1 条收款记录。
   - 新增、编辑或删除 1 条项目相关设置。
3. 回到账户中心，点击“本地恢复”。
4. 选择备份 A。
5. 查看备份摘要预览，记录 6 张业务表数量。
6. 点击“确认恢复”。
7. 在二次确认弹窗中再次点击“确认恢复”。
8. 等待恢复完成提示。

预期结果：

- 恢复前显示备份摘要。
- schemaVersion 相同的备份显示“确认恢复”按钮。
- 二次确认弹窗明确提示当前本机业务数据将被备份替换。
- 恢复成功后显示“恢复完成”。
- 恢复后 6 张业务表数量与备份摘要一致。
- 恢复后数据内容回到备份 A 的状态。

需要核对的 6 张业务表：

- `devices`
- `timing_records`
- `fuel_logs`
- `maintenance_records`
- `account_payments`
- `project_device_rates`

## 3. 自动备份测试

目标：验证真实恢复前会自动备份当前数据，并且自动备份失败时不会进入恢复事务。

正常自动备份场景：

1. 准备备份 A。
2. 修改当前本机数据，使当前数据与备份 A 不同。
3. 选择备份 A 执行恢复。
4. 恢复成功后查看成功提示。

预期结果：

- 恢复前 App 自动导出当前本机数据备份。
- 恢复成功提示中说明“恢复前已自动备份当前数据”。
- 不在 SnackBar 中展示过长文件路径。
- 如需展示路径，应使用弹窗或结果详情展示。

自动备份失败场景：

1. 使用测试环境模拟自动备份失败，例如临时让备份目录不可写，或通过测试注入让导出函数返回失败。
2. 选择有效备份文件并尝试恢复。

预期结果：

- App 提示“恢复前自动备份失败，已终止恢复，当前数据未被修改。”
- 不进入数据库恢复事务。
- 当前本机数据保持恢复前状态。
- 不出现部分数据被清空或部分数据被写入的情况。

记录项：

- 自动备份失败的模拟方式。
- 是否进入恢复事务。
- 当前数据是否保持不变。
- 自动备份路径展示策略是否符合预期。

## 4. schemaVersion 测试

目标：验证恢复功能严格执行 schemaVersion 策略。

测试项：

1. `schemaVersion == 当前版本`
   - 准备 `meta.schema_version = 9` 的备份。
   - 选择备份并预览。
   - 预期：允许预览，并显示“确认恢复”按钮。

2. `schemaVersion 低于当前版本`
   - 准备 `meta.schema_version < 9` 的备份。
   - 选择备份并预览。
   - 预期：允许预览，但不允许恢复；提示“当前版本暂不支持恢复旧版备份，请使用相同版本导出的备份。”

3. `schemaVersion 高于当前版本`
   - 准备 `meta.schema_version > 9` 的备份。
   - 选择备份。
   - 预期：拒绝恢复，并提示“备份文件版本较新，请升级 App 后再试。”

4. `schemaVersion 缺失`
   - 删除备份文件中的 `meta.schema_version`。
   - 选择备份。
   - 预期：判定为无效备份，提示备份文件格式不完整。

## 5. 非法文件测试

目标：验证非法文件不会进入恢复事务，也不会修改当前数据。

测试项：

- 非 JSON 文件。
- JSON 解析失败，例如内容为 `{ invalid json`。
- 顶层不是 Map，例如顶层为数组 `[]`。
- 缺少 `meta`。
- 缺少 `data`。
- 缺少某张业务表，例如删除 `data.devices`。
- 某张业务表不是 List，例如将 `data.devices` 改为 `{}`。
- 记录不是 Map，例如将 `data.devices` 的某一项改为字符串。
- 必要字段缺失，例如删除 `devices.id` 或 `timing_records.device_id`。

预期结果：

- App 显示用户可理解的错误提示。
- 不显示“确认恢复”按钮。
- 不调用真实恢复写入。
- 当前本机数据保持不变。

## 6. 事务回滚测试

目标：验证写入中途失败时，数据库事务会完整回滚，不出现部分恢复成功。

建议构造方式：

- 构造会导致写入失败的数据，例如：
  - `project_device_rates` 中主键冲突。
  - 必要字段类型错误。
  - 某条记录字段值违反数据库约束。

步骤：

1. 准备当前本机数据，并记录恢复前 6 张业务表数量。
2. 准备一个预检可通过但写入会失败的备份文件。
3. 执行恢复。
4. 查看失败提示。
5. 重新进入各页面核对数据。

预期结果：

- 恢复失败时提示“恢复失败，数据库已回滚，当前数据保持恢复前状态。”
- 恢复前自动备份如果已成功，应提示当前数据已自动备份。
- 当前本机 6 张业务表数量与恢复前一致。
- 不出现某些表已恢复、某些表未恢复的状态。

## 7. 重复点击 / 中断测试

目标：验证恢复过程中不会重复执行恢复。

测试项：

1. 恢复中重复点击确认按钮。
   - 预期：按钮进入 loading 或禁用状态，不能重复触发恢复。

2. 恢复中尝试关闭弹窗。
   - 预期：恢复中关闭按钮不可用，或不会中断正在进行的恢复事务。

3. 恢复中切到后台再回来。
   - 预期：App 不崩溃；恢复完成后显示成功或失败结果；不会重复执行恢复。

4. 恢复中快速切换页面。
   - 预期：不会出现 setState after dispose、重复弹窗或重复写入。

## 8. 页面刷新测试

目标：验证恢复成功后相关页面显示最新数据。

测试项：

1. 设备页刷新。
   - 恢复成功后返回设备页。
   - 预期：设备列表与备份 A 一致。

2. 计时页刷新。
   - 切换到计时页。
   - 预期：计时记录与备份 A 一致。

3. 油费页刷新。
   - 切换到油费页。
   - 预期：油费记录与备份 A 一致。

4. 维修页刷新。
   - 切换到维修页。
   - 预期：维修记录与备份 A 一致。

5. 账户/收款页刷新。
   - 切换到账户页。
   - 预期：收款记录和项目收款状态与备份 A 一致。

6. 项目相关设置刷新。
   - 打开项目详情或项目单价相关入口。
   - 预期：项目相关设置与备份 A 一致。

补充说明：

- 恢复成功后不刷新订阅状态。
- 恢复成功后不恢复 Pro 权限。
- 恢复成功后不恢复本地图片文件；如果自定义头像路径不存在，UI 应兜底显示默认头像。

## 9. 明确不测试或不支持的内容

以下内容不属于本轮本地覆盖恢复验收范围：

- 不测试合并恢复。
- 不测试追加恢复。
- 不测试部分表恢复。
- 不测试云端备份。
- 不测试云端恢复。
- 不测试多端同步。
- 不测试订阅状态恢复。
- 不测试 Pro 权限恢复。
- 不测试本地图片文件恢复。
- 不测试跨账号恢复。
