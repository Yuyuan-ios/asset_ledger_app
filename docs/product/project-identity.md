# 项目身份规则

## 稳定身份

- `projectId` 是项目的稳定身份。
- 业务归属、聚合、结算、统计应优先依赖 `projectId`。
- 编辑联系人或地址不应默认生成新的业务身份，除非对应流程明确要求重新归属。

## 展示与兼容字段

- `contact` 和 `site` 是展示字段，也是旧数据兼容字段。
- 它们可以用于标题、搜索、导入兼容和旧结构回填。
- 不应把 `contact/site` 当成比 `projectId` 更稳定的身份来源。

## 标题格式

- 普通项目标题：`联系人 · 地址`。
- 合并项目标题：`联系人 · 合并N项目`。

## 已有冻结规则

`docs/architecture/business_rules_v1.md` 仍记录 v1 阶段冻结业务规则。若本文件与冻结规则发生冲突，报告为 `待确认`，不要自行选择覆盖。

## 待确认

- 不同联系人、相同地址的合并展示细则待确认。
- 缺少联系人或地址时的降级标题格式待确认。
