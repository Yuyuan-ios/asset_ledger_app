# 外协的项目规则

## 用户可见文案

- 统一使用“外协的项目”。
- 旧文案如“项目外协”“外协项目”不作为新的用户可见默认文案。

## 文件扩展名

- 外协导入导出文件使用 `.jzt`。
- 不保留 `.jztshare` 兼容作为新功能默认行为。

## 数据结构优先级

- rich records 优先。
- legacy `export_lines` 只作为旧结构说明和兼容参考，不作为新功能默认结构。

## 实现边界

- 新功能应围绕 `.jzt` 和 rich records 设计。
- 旧结构兼容逻辑如需保留，应明确标注为兼容路径。
- 文案、导入导出、测试夹具需要使用同一套命名。

## 待确认

- `.jztshare` 历史文件是否需要独立迁移工具待确认。
